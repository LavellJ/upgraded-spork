# GDPR Overview - LearnOz

*General Data Protection Regulation compliance for EU students and families*

## Controller and Processor Roles

### Data Controller
**Primary Controller**: The school or educational institution deploying LearnOz
- Determines purposes and means of processing personal data
- Makes decisions about data collection, use, and retention
- Responsible for obtaining consent and fulfilling data subject rights
- Must have lawful basis for processing student personal data

### Data Processor  
**LearnOz Role**: Data processor providing educational technology services
- Processes personal data only on behalf of and under instructions from the school
- Implements appropriate technical and organizational security measures
- Assists controller in fulfilling data subject rights
- Notifies controller of any personal data breaches within 72 hours

## Personal Data We Process

### Student Learning Data
- **Progress Records**: Lesson completion, performance metrics, learning paths
- **Behavioral Analytics**: Time-on-task, engagement patterns, help-seeking behavior  
- **Educational Content**: Student-generated journal entries and reflection responses
- **Technical Data**: Device identifiers, session logs, error reports

### Instructor Data
- **Professional Information**: Teacher names, email addresses, class assignments
- **Usage Analytics**: Feature utilization, curriculum choices, student progress monitoring
- **Communication Records**: Support requests, feedback submissions, training completions

## Lawful Bases for Processing (Article 6)

### Public Task (Article 6(1)(e))
**Primary Basis**: Processing necessary for performance of educational tasks carried out in the public interest
- Delivering curriculum and instructional support
- Monitoring student progress and educational outcomes
- Supporting teacher professional development and instructional planning

### Legitimate Interests (Article 6(1)(f))
**Secondary Basis**: Processing necessary for legitimate interests (where balanced against data subject rights)
- **Security**: Protecting systems and data from unauthorized access
- **Service Improvement**: Analyzing usage patterns to enhance educational effectiveness
- **Technical Support**: Maintaining and troubleshooting educational technology services

### Consent (Article 6(1)(a))
**Supplementary Basis**: For optional features requiring explicit consent
- **Feedback Collection**: Voluntary satisfaction surveys and product improvement feedback
- **Extended Analytics**: Advanced learning analytics beyond core educational needs
- **Third-Party Integrations**: Optional connections to external educational tools

## Special Category Data (Article 9)

### Child Protection Measures
LearnOz is designed to **minimize** collection of special category data, but may occasionally process:
- **Learning Difficulties**: Implicit indicators from engagement patterns (no explicit diagnosis)
- **Accessibility Needs**: Screen reader usage, font size preferences, input accommodations

### Additional Safeguards
- **Data Minimization**: Collect only data necessary for educational purposes
- **Pseudonymization**: Use technical identifiers rather than direct personal identifiers where possible
- **Enhanced Security**: Additional encryption and access controls for sensitive educational data

## Data Subject Rights (Chapter III)

### Right of Access (Article 15)
**Individual Access**: Students/parents can request:
- Complete copy of all personal data processed
- Information about processing purposes, categories, and recipients
- Details about retention periods and automated decision-making
- **Response Time**: Within 1 month of verified request

### Right to Rectification (Article 16)
**Data Correction**: Students/parents can request:
- Correction of inaccurate learning progress data
- Completion of incomplete educational records
- Updates to outdated student information
- **Technical Corrections**: Immediate for system errors; educational review for academic disputes

### Right to Erasure (Article 17)
**"Right to be Forgotten"**: Available when:
- Personal data no longer necessary for educational purposes
- Student withdraws consent for optional processing
- Student graduates or transfers from school
- **Educational Limitations**: Core educational records may be retained per school policies

### Right to Restrict Processing (Article 18)
**Processing Limitation**: Students/parents can request restriction when:
- Accuracy of personal data is contested
- Processing is unlawful but erasure is not desired
- Data no longer needed but required for legal claims
- Objection to processing is pending verification

### Right to Data Portability (Article 20)
**Data Export**: Students/parents can receive:
- Learning progress data in structured, machine-readable format (JSON/CSV)
- Ability to transmit data to another educational technology provider
- **Educational Continuity**: Facilitates school transfers and technology transitions

### Right to Object (Article 21)
**Processing Objection**: Students/parents can object to:
- Processing based on legitimate interests (must be balanced against educational needs)
- Direct marketing (not applicable - LearnOz does not engage in marketing to students)
- Automated decision-making and profiling

## Children's Data Protection

### Age of Digital Consent
- **Under 13**: Always requires parental consent for any data processing
- **13-16**: May require parental consent depending on EU member state law
- **16+**: Can provide own consent for most processing activities

### Enhanced Protections
- **Privacy by Design**: Child-friendly interfaces and clear, simple language
- **Data Minimization**: Collect only data essential for educational objectives
- **Consent Management**: Clear mechanisms for withdrawing consent
- **Regular Review**: Periodic assessment of continued necessity for data processing

## Cross-Border Data Transfers

### EU/EEA Processing
**Preferred**: Personal data processed within EU/EEA when possible
- Cloud infrastructure in EU regions
- EU-based support and development teams
- Compliance with local data protection authority requirements

### Third Country Transfers
**If Required**: Transfers outside EU/EEA only with appropriate safeguards:
- **Adequacy Decisions**: To countries with EU-approved data protection levels
- **Standard Contractual Clauses**: EU-approved contract terms ensuring GDPR compliance
- **Binding Corporate Rules**: For transfers within multinational organizations

## Data Protection Impact Assessment (DPIA)

### High-Risk Processing Triggers
LearnOz may require DPIA when:
- **Large Scale Processing**: Deployment across multiple schools or districts
- **Automated Decision-Making**: AI-powered learning recommendations
- **Vulnerable Populations**: Processing children's data at scale
- **Behavioral Monitoring**: Detailed learning analytics and engagement tracking

### DPIA Components
- Systematic description of processing operations
- Assessment of necessity and proportionality
- Risk analysis for data subject rights and freedoms
- Measures to address identified risks

## Data Breach Response

### Breach Detection
- **Automated Monitoring**: Real-time alerts for unauthorized access attempts
- **Regular Audits**: Quarterly security assessments and penetration testing
- **Staff Training**: Recognition and reporting of potential security incidents

### Notification Requirements
- **Controller Notification**: LearnOz notifies school within 72 hours of breach discovery
- **Data Protection Authority**: School notifies relevant DPA within 72 hours if high risk
- **Data Subjects**: Individual notification if high risk to rights and freedoms
- **Communication**: Clear, plain language explanation of breach and protective actions

## Technical and Organizational Measures

### Security Controls
- **Encryption**: AES-256 encryption for data at rest and TLS 1.3 for data in transit
- **Access Controls**: Multi-factor authentication and role-based permissions
- **Network Security**: Firewalls, intrusion detection, and regular security monitoring
- **Data Backups**: Encrypted backups with geographic distribution

### Organizational Policies
- **Privacy by Design**: Data protection considerations in all development processes
- **Staff Training**: Regular GDPR awareness and data handling training
- **Vendor Management**: Due diligence and contractual protections for third-party services
- **Incident Response**: Documented procedures for security and privacy incidents
